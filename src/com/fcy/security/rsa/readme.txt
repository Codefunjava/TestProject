
RSA加密算法，较为典型的对称加密算法，基于数学问题中的大数因子分解难题。

算法思想：
	1.甲方构建密钥对（公钥-私钥），并将公钥公布给乙方。
	2.甲方发送数据给乙方：
	    甲方使用私钥加密数据，乙方收到数据后利用公钥解密数据。
	3.乙方发送数据给甲方：
	    乙方使用公钥加密数据，甲方收到数据后利用私钥解密数据。    
	
	由第二步和第三步可知，公钥和私钥都可以用来加密、解密数据，不过【公钥加
密的数据只能由私钥来解密，私钥加密的数据只能由公钥来解密】。
	
	这里说的公钥，应该不是公布出来，让全世界人民都知道的意思吧？
应该只是公布给乙方知道。。。
	
安全性：
	公钥是甲方发送给乙方的，其在传递过程中有可能被截获，因此存在一定的安全隐患，如果
要建立更安全的加密消息传递模型，就需要甲乙两方构建两套非对称加密算法密钥，仅遵循“私钥
加密，公钥解密”的方式进行加密消息传递。
